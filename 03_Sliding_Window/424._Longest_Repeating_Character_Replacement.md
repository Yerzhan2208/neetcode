## 🎯 Core Intuition
I was struggling with this problem, so I first looked at the hints, so obviously the solution should be related to sliding window. So the first step is to initialize the left and right side of this window. Then, we can calculate the frequency of each character and count it in the hash map. The next step is to check if other (non-frequent) characters are larger than k, if they are, we move the left side of the window and reduce the frequency of left character, if they are not, we continue to move right side and calculate the maximum result.


## 🚀 Optimal Solution

### Python Implementation
```python
class Solution:
    def characterReplacement(self, s: str, k: int) -> int:
        count = {}
        res = 0
        
        l = 0
        maxf = 0
        for r in range(len(s)):
            count[s[r]] = 1 + count.get(s[r], 0)
            maxf = max(maxf, count[s[r]])

            while (r - l + 1) - maxf > k:
                count[s[l]] -= 1
                l += 1
            res = max(res, r - l + 1)

        return res
```
